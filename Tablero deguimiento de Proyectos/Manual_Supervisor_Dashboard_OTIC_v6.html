¿Qué es el tablero y cómo funciona?

El Dashboard OTIC SDM v6.0 es un archivo HTML que se abre en el navegador (Chrome o Edge recomendado). Permite hacer seguimiento minucioso a contratos de la OTIC: el proveedor entrega un Excel cada semana, el supervisor lo carga al tablero, y el sistema genera KPIs, curva S, cronograma y alertas automáticamente.

Flujo general de trabajo

📁 Abrir tablero
→
✚ Crear proyecto
→
📊 Cargar Excel S01
→
🔒 BL congelada
→
💾 Guardar HTML
→
📊 Cargar Excel S02, S03…
→
💾 Guardar HTML
💡
Concepto clave — Línea Base (BL)

La primera vez que carga un Excel, el sistema congela la Línea Base: las fechas y duraciones originales del contrato. Todas las semanas siguientes comparan el avance real contra esa BL. La BL nunca cambia sola — solo un Control de Cambios aprobado puede modificarla.

📁
El tablero vive en un archivo HTML

Cada vez que hace cambios o carga un Excel, debe hacer clic en "Guardar HTML" para descargar la versión actualizada. Este archivo descargado contiene todos los datos embebidos — puede enviarlo por correo, abrirlo en cualquier computador, o compartirlo sin depender de internet ni de servidores.

Paso 2 — La plantilla Excel del proveedor

La plantilla Excel es el documento que el proveedor (contratista) llena cada semana y envía al supervisor. Tiene 4 hojas. El supervisor entrega la plantilla al proveedor al inicio del contrato — el proveedor la completa y la devuelve semanalmente.

📌
Número de actividades — es variable

La plantilla puede tener cualquier número de actividades según la complejidad del contrato. No está limitada a 17 filas. Puede tener 10, 30, 50 o más actividades. Lo importante es respetar los tipos de fila (PROYECTO, FASE, ACTIVIDAD, HITO) y no modificar las columnas de Línea Base.

Hoja 1 — INSTRUCCIONES_PMP

Explica la metodología de cálculo (método PMP, igual que MS Project). El proveedor puede consultarla pero no la modifica.

Hoja 2 — IDENTIFICACION

Contiene 3 secciones:

Sección¿Quién la llena?Campos
A. Datos del proyecto 🔒 No modificar Nombre, contrato, contratista, supervisor, fechas BL, valor
B. Información del reporte ★ ✏️ Proveedor llena
  • Período / Semana ★ — Ej: "Semana 04 — Agosto 2026"
  • Fecha de corte ★ — Fecha a la que reporta (DD/MM/AAAA). Ej: 30/08/2026
  • Responsable técnico ★ — Nombre e cargo del responsable
  • Correo de contacto ★ — Correo del responsable
C. Observaciones generales ★ ✏️ Proveedor llena Texto libre con: actividades del período, bloqueos activos, compromisos próxima semana. Aparecerá en la sección "Análisis a la fecha" del tablero.

Hoja 3 — AVANCE_ACTIVIDADES Principal

Es la hoja más importante. Contiene el cronograma completo. Las columnas están divididas en dos grupos:

Columna¿Quién la llena?Descripción
EDT🔒 No modificarCódigo jerárquico: 1, 1.1, 1.2, ★1.H…
Nombre de tarea🔒 No modificarNombre oficial de la actividad
Tipo🔒 No modificarPROYECTO / FASE / ACTIVIDAD / HITO
Comienzo 🔒🔒 No modificarFecha inicio de la Línea Base
Fin 🔒🔒 No modificarFecha fin de la Línea Base
Días🔒 No modificarDuración calculada
% esperado ★✏️ Proveedor llenaSolo en la fila PROYECTO (fila resumen). Porcentaje planeado según cronograma a la fecha de corte. Solo se llena en la fila del PROYECTO — el sistema calcula el resto.
% completado ★✏️ Proveedor llenaPorcentaje real de avance de cada ACTIVIDAD (0–100). No llenar en FASES ni HITOS — el sistema los calcula. No puede ser negativo ni mayor a 100.
Estado ★✏️ Proveedor llenaCompletado / En progreso / No Iniciado / Bloqueado
Observaciones ★✏️ Proveedor llenaNota breve de la actividad. Solo en ACTIVIDADES con avance o bloqueos.
⚠️
Reglas importantes para el proveedor al llenar AVANCE_ACTIVIDADES
  • NO modificar las columnas grises (Comienzo, Fin, Días): son la Línea Base congelada.
  • El % completado solo se llena en filas tipo ACTIVIDAD — nunca en FASE ni HITO.
  • Los valores deben estar entre 0 y 100. Un valor negativo o mayor a 100 genera error.
  • Si una actividad aún no inicia, dejar 0 (no en blanco).
  • Si una actividad cambia de fecha (por CC aprobado), actualizar las columnas Comienzo/Fin solo si el supervisor aprobó el CC previo.
  • Las filas de FASE e HITO se calculan automáticamente — no se deben llenar.

Hoja 4 — EVENTOS

El proveedor registra aquí eventos del período: riesgos, bloqueos, solicitudes de cambio (CC). Cada evento en una fila.

💡
Los eventos del Excel aparecen automáticamente en el tablero

Si el proveedor registra correctamente un evento de tipo TIEMPO, COSTO o ALCANCE en la hoja EVENTOS, el tablero lo auto-registra como Control de Cambios al importar. Los eventos de tipo RIESGO o BLOQUEO aparecen en la sección "Riesgos Activos" y en el panel de Gestión.

Paso 3 — Cargar el Excel en el tablero

Este proceso se repite cada semana cuando el proveedor entrega su reporte. Consta de 3 pasos dentro del tablero.

1

Abra el archivo HTML guardado de la semana anterior

Siempre trabaje sobre el HTML guardado la semana anterior (el que descargó con "Guardar HTML"), no sobre el archivo fuente original. El HTML guardado ya contiene todos los datos acumulados.

2

Haga clic en "✓ Carga Excel" en la barra superior

Se abre el modal de carga. Verá 3 pestañas en la parte superior: 1 · Cargar Excel → 2 · Validación → 3 · Confirmar.

3

Paso 1 — Cargar Excel: arrastre el archivo o haga clic para buscarlo

Seleccione el archivo .xlsx que envió el proveedor. El sistema lo leerá automáticamente y pasará al Paso 2.

4

Paso 2 — Validación: revise los resultados

El sistema valida los datos. Vea la Sección 5 para saber qué significa cada mensaje.

5

Paso 3 — Confirmar: haga clic en "✓ Importar y actualizar tablero"

Solo si la validación lo permite. El tablero se actualiza con los datos nuevos. Aparece un mensaje verde de confirmación (toast).

🚫
Primera carga = Línea Base (¡no repetir!)

La primera vez que carga el Excel de un contrato, el sistema congela la Línea Base. Las fechas del cronograma quedan bloqueadas permanentemente como referencia. A partir de la semana 2, el sistema solo actualiza el porcentaje real — las fechas BL no cambian solas.

5

Paso 4 — Interpretar la validación (Paso 2)

Después de cargar el Excel, el sistema analiza los datos y muestra varios paneles. Aprenda a leerlos:

✅ "XX filas OK. Sin errores críticos."

Todos los datos tienen formato correcto. Puede continuar a Confirmar.

✗ Error(es) — "Corrija los valores en el Excel y vuelva a cargar"

Hay datos inválidos. Los más comunes:

  • Valor negativo: una actividad reportó % menor a 0. El proveedor debe corregirlo a 0 mínimo.
  • Valor mayor a 100%: una actividad reportó más del 100%. El proveedor debe corregirlo a 100 máximo.
  • Fecha posterior al corte: una actividad reporta avance pero su fecha de inicio es futura.

Acción: devuelva el Excel al proveedor para corrección y vuelva a cargarlo.

⚠️ Advertencia(s) — "Revise; puede continuar si los valores son intencionales"

Hay datos sospechosos pero el sistema permite continuar si el supervisor lo aprueba. Ejemplos: actividad que reporta avance con fecha futura (puede ser intencional), padre con % diferente al promedio hijos. Puede continuar con precaución.

🔒 "Línea base bloqueada" — Auditoría de integridad

El sistema compara las actividades actuales con la BL original y detecta cambios de duración. No bloquea la importación — es informativo para el supervisor.

🚫 "Importación bloqueada — Cambio de fechas detectado"

El Excel tiene fechas de actividades diferentes a la BL congelada y no hay un CC de Tiempo aprobado. Acción requerida: registre primero un Control de Cambios de Tiempo (ver Sección 8) y luego vuelva a intentar la carga.

✅ "Cambio de fechas autorizado — CC de tiempo aprobado registrado"

Hay un CC de Tiempo registrado que cubre los cambios de fecha. La importación puede proceder normalmente.

Vista previa de datos

Antes de confirmar, el sistema muestra una tabla con las primeras actividades y sus valores. Verifique que los porcentajes parezcan razonables antes de hacer clic en Confirmar.

Paso 3 — Resumen antes de confirmar

El paso final muestra un resumen: número de actividades, eventos detectados, fecha de corte, % real del campo resumen del Excel. Si todo se ve bien, haga clic en "✓ Importar y actualizar tablero".

Panel 02 — Curva S & Gestión (pestañas)

  • Curva S & Gestión: Gráfico de avance real vs planeado semana a semana. El punto azul es el corte actual. Al pasar el cursor sobre un punto ve el valor. La línea naranja es la BL planeada — la azul es el avance real.

  • Ficha contractual: Datos del contrato. Se llena al crear el proyecto.
  • Cambios: Historial de todos los Controles de Cambio registrados (CC1, CC2…) con tipo, descripción, documento y trazabilidad de fechas.

Paso 6 — Guardar el HTML (¡CRÍTICO!)

🚫
Si no guarda, pierde todo el trabajo de la sesión

El tablero es un archivo HTML que se abre en el navegador. Si cierra la ventana sin guardar, o presiona F5 (recargar), los cambios y datos cargados en esa sesión se pierden. Siempre guarde al final de cada sesión.

1

Haga clic en "💾 Guardar HTML" en la barra superior

El navegador descarga un archivo con nombre como: Dashboard_OTIC_v6_20260521_1proy.html. El nombre incluye la fecha del día.

2

Guarde el archivo en una carpeta organizada

Se recomienda una carpeta por contrato, con subcarpetas por semana. Ejemplo: CTO2026-0001 / Semana04 / Dashboard_S04.html

3

La próxima semana, abra el archivo guardado (no el original)

El archivo guardado contiene todos los datos embebidos. Al abrirlo, el tablero carga directamente sin preguntar nada — todos los proyectos, CCs e historial están ahí.

💡
Backup automático en el navegador

El tablero guarda automáticamente en el navegador (localStorage) cada vez que importa o registra algo. Si abre el archivo fuente original y hay datos guardados en el navegador, el sistema preguntará: "¿Desea restaurar los proyectos guardados?" — haga clic en Aceptar para recuperarlos. Luego guarde el HTML inmediatamente.

Paso 7 — Control de Cambios (CC)

El módulo de Control de Cambios permite registrar modificaciones aprobadas al contrato. Solo registre CCs que cuenten con acto administrativo aprobado (otrosí, acta de suspensión, resolución). Hay 3 tipos:

TipoCuándo usarloEfecto en el tablero
⏱ Tiempo
Prórroga / Suspensión		 Cuando el contrato se amplía o suspende. Requiere nueva fecha fin. Actualiza la Fecha Fin Contrato en todos los KPIs y el cronograma.
💰 Costo
Adición / Reducción		 Cuando el valor del contrato cambia. Actualiza el Valor del Portafolio y la ficha contractual.
📐 Alcance
Actividades / Entregables		 Cuando se agregan, eliminan o restructuran actividades. Registra el cambio en el historial. Las actividades se actualizan en el próximo Excel.

Cómo registrar un CC de Tiempo (prórroga)

1

Haga clic en "⚡ Control cambios" en la barra superior

Se abre el modal con la pestaña "+ Registrar cambio".

2

Seleccione el tipo "Tiempo"

Aparecerá la fecha fin actual pre-llenada automáticamente. Solo debe ingresar la nueva fecha fin.

3

Complete el formulario

CampoEjemplo
Nueva fecha fin ★15/12/2026 (formato DD/MM/AAAA)
Descripción / Título ★Prórroga 45 días por retraso en integración
Causa / OrigenSincronización con sistemas legados tomó más tiempo
Documento soporte ★Otrosí No. 2
Aprobado porSupervisor OTIC / Nombre del funcionario
Fecha de aprobación ★01/09/2026

El sistema calculará los días de prórroga automáticamente al seleccionar la nueva fecha.

4

Haga clic en "✓ Registrar cambio" y confirme el diálogo

El sistema muestra un resumen: "Este CC extiende la fecha fin en X días". Haga clic en Aceptar.

5

Verifique: la fecha fin del contrato se actualiza en toda la interfaz

El KPI "Fecha Fin Contrato", el panel izquierdo y el cronograma muestran inmediatamente la nueva fecha. En la pestaña "Cambios" aparecerá el nuevo CC con número consecutivo (CC1, CC2…).

⚠️
CC requerido antes de reimportar si el Excel cambió fechas

Si el proveedor entrega un Excel con fechas de actividades diferentes a la BL, el tablero bloqueará la importación con un panel rojo. Debe registrar primero el CC de Tiempo aprobado, y luego volver a intentar la carga. El sistema reconocerá el CC y permitirá la importación.

Historial de cambios

En la pestaña "Cambios" del panel central puede ver todos los CCs registrados, con su tipo, descripción, documento soporte, fecha y la trazabilidad visual de la fecha anterior tachada y la nueva fecha resaltada. El contador en la parte superior muestra cuántos CCs hay por tipo.

Paso 8 — Portafolio Gerencial

El tablero incluye dos vistas de portafolio para gestión gerencial:

Consolidador Maestro

Acceso: botón "Consolidar" en la barra superior. Permite importar varios archivos HTML de diferentes supervisores y consolidar los KPIs de todos los contratos en una sola vista. Cada supervisor exporta su dashboard con "Guardar HTML" y lo envía. El consolidador une todos.

Vista Gerencial (PIN protegida)

Acceso: teclado Shift + Ctrl + G en cualquier parte del tablero. Solicita un PIN de acceso. Muestra:

  • Resumen del portafolio: proyectos activos, en riesgo, con alertas
  • Avance promedio ponderado del portafolio
  • Valor total del portafolio ejecutado vs total
  • Semáforo RAG (Rojo/Ámbar/Verde) por proyecto
  • Hitos próximos de todos los proyectos (90 días)
  • Alertas que requieren decisión gerencial (escalamiento)
💡
El sistema genera alertas automáticas de escalamiento

Cuando un proyecto tiene SPI menor a 0.85 o brecha superior al -10%, el portafolio genera automáticamente una alerta: "Considere reunión de escalamiento". Esto ayuda a identificar proyectos que requieren intervención inmediata sin revisar cada uno manualmente.

Proceso semanal resumido

Una vez el proyecto está activo y la Línea Base establecida, el proceso semanal es:

1

Recibir el Excel del proveedor

El proveedor llena las columnas amarillas (% esperado, % completado, Estado, Observaciones) y las hojas IDENTIFICACION y EVENTOS, y envía el archivo.

2

Abrir el HTML guardado de la semana anterior

Doble clic en el archivo descargado la semana pasada. Carga directamente con todos los datos históricos.

3

Si hay CC aprobados pendientes, registrarlos primero

Antes de importar el Excel, registre cualquier CC de Tiempo, Costo o Alcance que haya sido aprobado en el período.

4

Cargar el Excel del proveedor ("✓ Carga Excel")

Revisar la validación, corregir si hay errores, confirmar la importación.

5

Revisar los KPIs y el tablero actualizado

Verificar avance, brecha, SPI, hitos próximos, riesgos activos y análisis del proveedor.

6

Hacer clic en "💾 Guardar HTML"

Descargar el archivo actualizado y guardarlo en la carpeta de seguimiento del contrato.

⏱️
Tiempo estimado por sesión semanal: 10–15 minutos

Una vez familiarizado con el proceso, la carga semanal toma muy poco tiempo. La mayoría del tiempo es revisar la información que el sistema genera automáticamente.

Errores frecuentes y soluciones

SíntomaCausa probableSolución
El tablero muestra "Sin proyectos activos" al abrir Abrió el archivo fuente original en lugar del HTML guardado Abra el archivo descargado con "Guardar HTML". Si el navegador pregunta por restaurar backup, haga clic en Aceptar.
"Importación bloqueada — Cambio de fechas detectado" El Excel tiene fechas diferentes a la BL y no hay CC de Tiempo registrado Registre primero el CC de Tiempo en "⚡ Control cambios", luego vuelva a cargar el Excel.
Error: "Valor negativo en actividad X" El proveedor ingresó un % negativo Devuelva el Excel al proveedor. El valor debe ser entre 0 y 100. Vuelva a cargar el Excel corregido.
Error: "Valor >100% en actividad X" El proveedor ingresó un % mayor a 100 Devuelva el Excel. El máximo es 100%. Vuelva a cargar el Excel corregido.
Los KPIs no cambian después de importar No se hizo clic en "✓ Importar y actualizar tablero" en el Paso 3 Repita la carga y asegúrese de llegar al Paso 3 y confirmar.
Pérdida de datos al presionar F5 o recargar Se recargó el archivo fuente original (que siempre inicia vacío) No presione F5. Trabaje siempre desde el HTML guardado. Si hay backup en el navegador, use el diálogo de restauración.
La fecha de prórroga no se calcula al ingresar fecha fin Formato de fecha incorrecto Use siempre el formato DD/MM/AAAA (ej: 15/12/2026). No use AAAA-MM-DD.
El campo "Días de prórroga" queda vacío en el CC No se seleccionó la fecha o se ingresó en formato incorrecto Use el selector de fecha del campo o escriba en formato DD/MM/AAAA. El cálculo es automático.
La curva S muestra el planeado diferente al KPI El tooltip de la curva muestra el planeado en un punto de tiempo anterior a la fecha de corte Normal. Pase el cursor hasta el punto correspondiente a la fecha de corte del Excel — debe coincidir con el KPI.
El archivo Excel no sube / error de lectura El archivo está abierto en Excel o tiene formato incorrecto Cierre el archivo en Excel antes de subirlo. Use archivos .xlsx (no .xls ni .csv).
